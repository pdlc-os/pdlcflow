Onboarding · one-page map

The pdlcflow journey

From nothing installed to shipping a feature — and where each onboarding doc fits. Drive it all by typing slash-commands in the Studio chat; it runs autonomously and pauses at gates for you.

1

Install & open the Studio

One command. No clone, no API keys to start.

bash -c "$(curl -fsSL …/deploy/install.sh)"
  • Answer a short setup wizard (version · login? · real LLM? · evals?).
  • Studio UI → localhost:8080  ·  API health → localhost:8000/health.
  • The LLM runs an offline stub until you wire a provider — explore freely.

→ 2 · Getting started · 2a · Setup walkthrough

2

Pick your road

Where you start from decides your first move.

🌱 Greenfield — brand-new product

Little/no code, no prior roadmap.

  • Run /init — author Constitution · Intent · seed Roadmap.
  • Ends at the init_approve gate → into Inception.
  • Then work features (step 3).

→ have a spec?

🏛️ Brownfield — existing app

Bring your code, history & roadmap.

  • Upstream pdlc project → pdlc-migrate scan→push→taxonomy→backfill. Dashboards fill day one.
  • Any git repo → connect it + /init to seed your roadmap.

→ 5 · Bringing your own roadmap
